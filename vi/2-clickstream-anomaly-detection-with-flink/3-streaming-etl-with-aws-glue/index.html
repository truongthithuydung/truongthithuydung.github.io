Xử lý ETL thời gian thực với AWS Glue

🎓 Giới Thiệu

Trong lab này, bạn sẽ học cách nhập, xử lý, và tiêu thụ dữ liệu streaming sử dụng các dịch vụ serverless của AWS như: Kinesis Data Streams, Glue, S3, Athena.

Để mô phỏng dữ liệu streaming đầu vào, chúng ta sẽ sử dụng Kinesis Data Generator (KDG).

⚡ Thiết Lập Kinesis Stream

  1. Truy cập vào AWS Kinesis console.

  1. Nhấp vào Create data stream.

  2. Điền thông tin:

    • Stream name: TicketTransactionStreamingData
    • Capacity mode: Provisioned
    • Provisioned shards: 2

  1. Nhấp vào Create data stream. ✔️

📝 Hướng Dẫn Lab

🔑 Tạo Bảng cho Nguồn Dữ Liệu Kinesis Stream trong Glue Data Catalog

  1. Truy cập vào AWS Glue console.
  2. Trong menu AWS Glue, chọn Data Catalog → Tables và nhấp vào nút Add table.

  1. Nhập TicketTransactionStreamData làm tên bảng.

  1. Nhấp vào Create database, đặt tên là tickettransactiondatabase, và nhấp vào Create.

  1. Từ dropdown, chọn tickettransactiondatabase và nhấp vào Next.

  1. Chọn Kinesis làm nguồn, sau đó:
    • Chọn khu vực AWS phù hợp.
    • Chọn TicketTransactionStreamingData từ dropdown stream.
    • Đặt JSON làm định dạng dữ liệu đầu vào.

  1. Để schema trống (chúng ta sẽ bật phát hiện schema trong bước tiếp theo) và nhấp vào Next.

  1. Xem lại các cài đặt và nhấp vào Create.

  1. Nhấp vào bảng để xem các thuộc tính của nó.

💼 Tạo và Kích Hoạt Job Glue Streaming

  1. Trong Glue Console, nhấp vào Data Integration → ETL jobs.

  1. Nhấp vào Create job from a blank graph.
  2. Chọn Amazon Kinesis làm nguồn.

  1. Nhấp vào nút Amazon Kinesis trên canvas, sau đó cấu hình như sau:
    • Name: Amazon Kinesis
    • Source: Data Catalog table
    • Database: tickettransactiondatabase
    • Table: tickettransactionstreamdata
    • Đảm bảo Detect schema được chọn.
    • Để các trường khác ở mặc định.

  1. Nhấp vào nút + để thêm một node. Sau đó, chọn Amazon S3 trong tab Targets.

  1. Nhấp vào Data target - S3 bucket trên canvas, và cấu hình như sau:
    • Format: Parquet
    • Compression Type: Uncompressed
    • S3 Target Location: Duyệt đến S3, chọn bucket có tên chứa dmslabs3bucket, và thêm TicketTransactionStreamingData/ vào đường dẫn S3.
      • Đường dẫn sẽ trông như: s3://xxx-xxx-dmslabs3bucket-xxx/TicketTransactionStreamingData/ (đảm bảo có dấu / ở cuối).

  1. Dưới tab Job details, cấu hình như sau:
    • Name: TicketTransactionStreamingJob
    • IAM Role: Chọn xxx-GlueLabRole-xxx từ dropdown.
    • Type: Spark Streaming

  1. Nhấn Save để tạo job.
  2. Sau khi job được tạo thành công, nhấp vào Run để bắt đầu xử lý dữ liệu. 🔄

🎥 Kích Hoạt Dữ Liệu Streaming từ KDG

  1. Khởi Động KDG

    • Mở KDG bằng URL bạn đã lưu từ khi thiết lập CloudFormation.
    • Đăng nhập bằng tên người dùng và mật khẩu bạn đã cung cấp khi triển khai CloudFormation stack.

  2. Chọn Khu Vực và Kinesis Stream

    • Đảm bảo bạn đã chọn khu vực phù hợp.
    • Từ danh sách dropdown, chọn TicketTransactionStreamingData làm Kinesis stream mục tiêu.
    • Để Records per second ở giá trị mặc định (100 records per second).
    • Đối với record template, nhập NormalTransaction làm tên payload, và sao chép template payload như sau:
    {
 "customerId": "{{random.number(50)}}",
 "transactionAmount": {{random.number(
     {
         "min":10,
         "max":150
     }
 )}},
 "sourceIp" : "{{internet.ip}}",
 "status": "{{random.weightedArrayElement({
     "weights" : [0.8,0.1,0.1],
     "data": ["OK","FAIL","PENDING"]
     }        
 )}}",
"transactionTime": "{{date.now}}"      
 }

  1. Gửi Dữ Liệu:
    Nhấp vào Send data để kích hoạt dữ liệu giao dịch mô phỏng. 🚀

🛠️ Tạo Glue Crawler cho Dữ Liệu Đã Biến Đổi

  1. Điều hướng đến AWS Glue console.
  2. Trong menu AWS Glue, chọn Crawlers và nhấp vào Create crawler.

  1. Đặt tên cho crawler là TicketTransactionParquetDataCrawler và nhấp vào Next.

  1. Nhấp vào Add a datasource.

  1. Chọn S3 làm kho dữ liệu và nhấp vào Browse S3.
    • Chọn bucket chứa dmslabs3bucket trong tên.
    • Chọn thư mục TicketTransactionStreamingData và nhấp vào Choose.
    • Sau khi cấu hình S3 bucket và đường dẫn dữ liệu, xem lại cài đặt và nhấp vào Add an S3 data source.

  1. Vì không cần nguồn dữ liệu khác, nhấp vào Next.
  2. Từ danh sách dropdown, chọn IAM role đã tồn tại có tên chứa GlueLabRole, sau đó nhấp vào Next.

  1. Chọn tickettransactiondatabase làm cơ sở dữ liệu đầu ra.
    • Đặt parquet_ làm tiền tố cho bảng.
    • Nhấp vào Next.
    • Đặt crawler chạy mỗi giờ để đảm bảo các phân vùng mới được thêm vào.

  1. Xem lại cấu hình crawler và nhấp vào Create crawler.

  1. Sau khi crawler được tạo, chọn crawler và nhấp vào Run crawler để kích hoạt lần chạy đầu tiên.

  1. Chờ cho đến khi crawler hoàn tất công việc (trạng thái sẽ hiển thị là “Completed”).

🚨 Kích Hoạt Dữ Liệu Giao Dịch Bất Thường từ KDG

  1. Giữ cho dữ liệu KDG đang chạy:
    • Mở một cửa sổ trình duyệt khác và khởi chạy KDG bằng URL bạn đã lưu từ khi thiết lập CloudFormation.
    • Đăng nhập bằng tên người dùng và mật khẩu bạn đã cung cấp khi triển khai CloudFormation stack.

  1. Cấu hình dữ liệu streaming:
    • Đảm bảo bạn đã chọn khu vực phù hợp.
    • Từ danh sách dropdown, chọn TicketTransactionStreamingData làm Kinesis stream mục tiêu.
    • Đặt Records per second là 1 (để mô phỏng một giao dịch mỗi giây).

  1. Chuẩn bị payload giao dịch bất thường:

    • Chọn Template 2 cho record template.
    • Đặt payload name là AbnormalTransaction.
    • Sao chép template payload sau vào trình soạn thảo:
    {
"customerId": "{{random.number(50)}}",
"transactionAmount": {{random.number(
    {
        "min":10,
        "max":150
    }
)}},
"sourceIp" : "221.233.116.256",
"status": "{{random.weightedArrayElement({
    "weights" : [0.8,0.1,0.1],
    "data": ["OK","FAIL","PENDING"]
    }        
)}}",
"transactionTime": "{{date.now}}"      
}
  • Click “Send data” to trigger the abnormal transactions (1 transaction per second, all from the same source IP address).

🔍 Phát Hiện Giao Dịch Bất Thường bằng Truy Vấn Ad-Hoc từ Athena

  1. Điều hướng đến AWS Athena console:

    • Nếu đây là lần đầu tiên bạn sử dụng Athena, mở rộng phần Get Started using Athena bên dưới.

  2. Chọn nguồn dữ liệu và cơ sở dữ liệu:

    • Chọn AwsDataCatalog làm nguồn dữ liệu.
    • Chọn tickettransactiondatabase làm cơ sở dữ liệu.
    • Làm mới để đảm bảo bảng parquet_tickettransactionstreamingdata xuất hiện trong danh sách.

  1. Truy vấn số lượng giao dịch theo địa chỉ IP nguồn:
    • Sao chép truy vấn sau để đếm số giao dịch từ mỗi source IP trong vòng một giờ qua: 
      SELECT count(*) as numberOfTransactions, sourceip
FROM "tickettransactiondatabase"."parquet_tickettransactionstreamingdata"
WHERE ingest_year='2024'
AND cast(ingest_year as bigint)=year(now())
AND cast(ingest_month as bigint)=month(now())
AND cast(ingest_day as bigint)=day_of_month(now())
AND cast(ingest_hour as bigint)=hour(now())
GROUP BY sourceip
ORDER BY numberOfTransactions DESC;
  • Thực thi truy vấn. Bạn sẽ thấy một số lượng lớn giao dịch đến từ cùng một địa chỉ IP nguồn.
