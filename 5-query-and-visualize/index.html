Athena and Quicksight

üîç Query Data with Amazon Athena

Step 1: Open Amazon Athena

  1. In the AWS Services Console, search for Athena.
  2. If this is your first time using Athena in your AWS Account, expand “Get Started using Athena” and follow the setup instructions.

Step 2: Select Database and Inspect Tables

  1. In the Query Editor, select your newly created database (e.g., "ticketdata").
  2. Click on the table “parquet_sporting_event_ticket” to inspect its fields.
    • The data types for id, sporting_event_id, and ticketholder_id should be (double).

Step 3: Query Across Multiple Tables

Next, we will query across the tables:

  • parquet_sporting_event
  • parquet_sport_team
  • parquet_sport_location

üìå Copy and run the following SQL query in the Athena Query Editor:

SELECT
    e.id AS event_id,
    e.sport_type_name AS sport,
    e.start_date_time AS event_date_time,
    h.name AS home_team,
    a.name AS away_team,
    l.name AS location,
    l.city
FROM parquet_sporting_event e,
     parquet_sport_team h,
     parquet_sport_team a,
     parquet_sport_location l
WHERE
    e.home_team_id = h.id
    AND e.away_team_id = a.id
    AND e.location_id = l.id;

Step 4: Create a View

  1. Click Create and then select View from query.
  2. Name the view sporting_event_info and click Create.

  1. Your new view is now created.

Step 5: Query the View

  1. Open a new query tab in Athena.
  2. Copy and paste the following SQL query:
SELECT 
    t.id AS ticket_id,
    e.event_id,
    e.sport,
    e.event_date_time,
    e.home_team,
    e.away_team,
    e.location,
    e.city,
    t.seat_level,
    t.seat_section,
    t.seat_row,
    t.seat,
    t.ticket_price,
    p.full_name AS ticketholder
FROM sporting_event_info e,
     parquet_sporting_event_ticket t,
     parquet_person p
WHERE
    t.sporting_event_id = e.event_id
    AND t.ticketholder_id = p.id;

Step 7: Save the Query

  1. Click on Save as.
  2. Give this query a name: create_view_sporting_event_ticket_info and some description.
  3. Click on Save query.

  1. Back to the query editor, you will see the query name changed.

  1. Now, click on Run.
  2. The results appear beneath the query window.

Step 8: Create Another View

  1. Click View from query.
  2. Name the view sporting_event_ticket_info.
  3. Click Create.

Step 9: Copy and Run the Query

  1. Copy the following SQL syntax into the Query 5 tab:
SELECT
    sport,
    count(distinct location) as locations,
    count(distinct event_id) as events,
    count(*) as tickets,
    avg(ticket_price) as avg_ticket_price
FROM sporting_event_ticket_info
GROUP BY 1
ORDER BY 1;

Create QuickSight Charts

This section guides you through different chart types in QuickSight.

Step 1: Populate the Chart

  1. In the Fields list, click the ticket_price column to populate the chart.
  2. Click the expand icon in the corner of ticket_price field.
  3. Select Show as Currency to display the number in dollar value.

Step 2: Add a Visual

  1. Click the Add button from the Visuals pane.
  2. In the Visual types area, choose the Vertical bar chart icon.
  3. Set the X-axis:
    • In the Fields list, select event_date_time.
    • The visualization should update automatically.
  4. Set the Y-axis (Value):
    • Select ticket_price from the Fields list.

Step 3: Adjust and Group Data

  1. Drag and move visuals to adjust space in the dashboard.
  2. In the Fields list, click and drag seat_level to the Group/Color box.
  3. Use the slider below the X-axis to fit all the data.

Step 4: Change the Chart Type

  1. In the Visuals pane, click Vertical bar chart.
  2. Under Change visual type, choose the Clustered bar combo chart icon.
  3. In the Fields list, click and drag ticketholder to the Lines box.
  4. In the Lines box:
    • Click the dropdown box.
    • Choose Aggregate: Count Distinct for Aggregate.
    • The Y-axis should update on the right-hand side.

Create QuickSight Parameters

In this section, we will create parameters with controls for the dashboard and assign them to a filter for all visuals.

Step 1: Open Parameters

  1. From the Toolbar, select the Parameters icon.

Step 2: Create the EventFrom Parameter

  1. Click Add to create a new parameter.
  2. For Name, type EventFrom.
  3. For Data type, choose Datetime.
  4. For Time granularity, set Hour.
  5. For Default value, select a date from the calendar corresponding to the earliest event_date_time in your graph.
    • Example: 2021-01-01 00:00.
  6. Click Create, then close the Parameter Added dialog box.

Step 3: Create the EventTo Parameter

  1. Click Add to create another parameter.
  2. Set the following attributes:
    • Name: EventTo
    • Data type: Datetime
    • Time granularity: Hour
    • Default value: Select a date from the calendar corresponding to the latest event_date_time in your graph.
      • Example: 2022-01-01 00:00.
  3. Click Create.

Step 4: Add Controls

  1. In the next window, you can select any option to perform operations with the parameter.
  2. Alternatively, click the three dots for the EventFrom parameter and choose Add control.
  3. For Display name, specify Event From and click Add.

  1. Repeat the process to add a control for EventTo, setting the Display name as Event To.

Now, you should see and expand the Controls section above the chart.

Create a QuickSight Filter

To finalize the process, we will connect a filter to the controls for all visuals.

  1. From the Toolbar, choose Filter.
  2. Click the plus icon (+) to add a filter for the field event_date_time.

  1. Click the newly added filter to edit its properties.

  1. Set Applied To dropdown to All applicable visuals. \
  2. For Filter type, choose Date & Time range.
  3. Set Condition as Between.
  4. Select Use Parameter, click Yes to apply to all visuals.
  5. Set Start date parameter to EventFrom.
  6. Set End date parameter to EventTo.

  1. Click Apply.

Add Calculated Fields

Now, let’s add calculated fields for day of the week and hour of the day, then visualize them using a scatter plot.

  1. Click the Data menu and select Add Calculated Field.

  1. Name the field: event_day_of_week.
  2. Enter the formula: extract(‘WD’, {event_date_time})
  3. Click Save

  1. Add another calculated field with the following attributes:
  • Calculated field name: event_hour_of_day
  • Formula: extract(‘HH’,{event_date_time})
  1. Click Visualize icon from the Tool bar and choose Add visual.

  1. For field type, select the scatter plot.

  2. In the Fields list, click the following attributes to set the graph attributes:

    • X-axis: “event_hour_of_day”
    • Y-axis: “event_day_of_week”
    • Size: “ticket_price”

You can now click on Publish on the top right corner of screen to publish your visuals. You can also, share it by clicking on the Share menu.
